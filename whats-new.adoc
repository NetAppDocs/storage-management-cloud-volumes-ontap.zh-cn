[[introducing-simplified-onboarding-of-cloud-volumes-ontap-from-aws-marketplace]]
=== 简化从 AWS Marketplace 载入 Cloud Volumes ONTAP 的流程

NetApp 为来自 AWS Marketplace 的 Cloud Volumes ONTAP 引入了简化的入门流程，减少了手动步骤和入门阻力。

以前，从 AWS Marketplace 部署 Cloud Volumes ONTAP 需要在多个屏幕之间导航，并手动配置角色、策略和凭据。这使得入门流程缓慢且容易出错。

新的入职流程通过以下功能简化了从市场订阅到全面运行的 Cloud Volumes ONTAP 环境的过渡：

. 订阅 AWS Marketplace 上的专用 Cloud Volumes ONTAP 列表：您在 AWS Marketplace 上订阅 Cloud Volumes ONTAP 独立列表，购买订阅并设置您的 AWS 帐户（如果您还没有）。
. 在 NetApp Console 上的引导路径：您在 NetApp Console 中设置帐户，Console 会自动引导您进入部署过程，并在右侧显示引导路径。如果您需要离开 Console，可以稍后继续引导行程。
. 关联您的订阅：您可以自动将订阅与 Console 关联，从而避免手动链接的步骤。
. 使用 AWS 凭据配置的控制台代理部署：您可以自动创建所需的 AWS 角色。该流程在您的 AWS 账户中部署 AWS CloudFormation 堆栈，其中包含供您查看的角色和策略，然后将其与 NetApp Console 相关联，从而无需手动设置 IAM。然后继续完成代理配置、网络和安全组设置，以开始代理部署。
. 在 AWS 中部署新的 Cloud Volumes ONTAP 系统：您可以直接导航到 `Quick create`向导，以实现更快、更简化的 Cloud Volumes ONTAP 部署。

[[support-for-next-generation-google-cloud-vms-in-cloud-volumes-ontap-ha]]
=== 支持 Cloud Volumes ONTAP HA 中的下一代 Google Cloud VM

借助 9.19.1，NetApp 将 Cloud Volumes ONTAP 高可用性 (HA) 部署从 N2 虚拟机迁移到下一代 Google Cloud C3 系列虚拟机，以获得更快、更具可扩展性的体验。


NOTE: 此支持仅适用于单可用区 (AZ) HA 部署。

现在，您可以在 Google Cloud 中部署 Cloud Volumes ONTAP 9.19.1 及更高版本时利用 C3 系列虚拟机。这些机器具有更高的性能和更大的容量限制。它们使用 Google Virtual NIC (gVNIC) 和 Hyperdisk Balanced 磁盘，在密集型工作负载上实现动态性能。

如果您的 Cloud Volumes ONTAP 系统运行 9.19.1 或更高版本，则用于在单个 AZ 中轻松部署的预配置包将自动使用 C3 VM，同时使您能够根据工作负载需求自定义 IOPS 和吞吐量参数。同样，在创建聚合时，您可以添加 Zonal Hyperdisk Balanced 磁盘，以在 Google Cloud 中实现更好的性能和可扩展性。您还可以选择 C3 系列机器的 LSSD 变体，以获得默认 Flash Cache 支持。

请注意以下几点：

* 在 Cloud Volumes ONTAP 部署中，您无法将 C3 虚拟机更改为 N2 系列虚拟机，反之亦然。但是，您可以在 C3 系列中的不同变体之间切换实例类型。
+

NOTE: 根据连接的数据磁盘数量，更改 C3 系列中的实例类型可能需要更长时间。请提前规划您的配置，以避免对性能造成影响。

* 更改实例类型时，如果选择 c3-standard-44 或 c3-standard-88 及其本地 SSD (LSSD) 变体作为目标 VM，则会自动应用 Google 网络服务 egress network tier _Tier-1_。如果切换回任何其他 C3 变体，则出口网络层将设置为 _Standard_。
+

NOTE: 将出口网络层更改为 _Tier-1_ 会产生额外费用。请相应地规划部署和升级。

* 如果选择 C3 的 LSSD 变体，默认情况下会启用 Flash Cache。即使目标变体支持 Flash Cache，您也无法在 Cloud Volumes ONTAP 部署中的 C3 LSSD 变体之间切换。
* 在 C3 虚拟机上，您可以向新的和现有的聚合添加卷。
* 将 N2 VM 类型的 Cloud Volumes ONTAP 系统复制到 C3 VM 类型时，磁盘类型默认设置为 Hyperdisk Balanced。
* 通过 NetApp Console 或使用 `PUT /gcp/ha/aggregates/{workingEnvironmentId}/{aggregateName}` API 添加聚合时，可以更改 IOPS 和吞吐量设置。


https://docs.netapp.com/us-en/cloud-volumes-ontap-relnotes/reference-configs-gcp.html["Google Cloud 中 Cloud Volumes ONTAP 支持的配置"^]

https://docs.cloud.google.com/compute/docs/general-purpose-machines#c3_series["Google 文档：C3 机器系列"^]

[[introducing-access-roles-for-cloud-volumes-ontap]]
=== Cloud Volumes ONTAP 访问角色简介

NetApp 为 Cloud Volumes ONTAP 引入了身份和访问管理 (IAM) 角色，实现了基于角色的访问控制 (RBAC)，以提升安全性并简化合规管理。用户可以继续使用其现有凭据登录 NetApp Console，以执行 Cloud Volumes ONTAP 操作。但是，身份验证现在通过身份联合进行路由，使 Console 管理员能够对这些功能进行更精细的控制。每个用户必须具有以下角色之一，才能对 Cloud Volumes ONTAP 执行任何操作。如果没有角色，用户将无法查看或访问 Cloud Volumes ONTAP 实例。

* *Cloud Volumes ONTAP viewer*：授予对 Cloud Volumes ONTAP 屏幕和选项卡的只读访问权限。此角色中禁用了某些屏幕和操作按钮。
* *Cloud Volumes ONTAP operator*：启用日常管理任务，例如添加卷、管理存储和配置复制。
* *Cloud Volumes ONTAP admin*：提供对所有 Cloud Volumes ONTAP 操作的完全访问权限。


Console 管理员从*管理 > 身份*为 Cloud Volumes ONTAP 用户分配角色。角色权限基于用户的组织，并应用于该组织中的所有 Cloud Volumes ONTAP 系统。

Cloud Volumes ONTAP 使用每个用户权限的本地缓存强制执行 RBAC，该缓存每小时自动刷新一次。当为用户分配新角色或更改其现有角色时，该更改可能需要最多一小时才能生效，此后，角色允许的所有工作流将被启用。


NOTE: 首次部署 Console 代理或将现有 Console 代理升级到 Console 4.9.0 或更高版本时，一次性种子设定过程会自动将 Cloud Volumes ONTAP 角色分配给组织中的所有现有用户。此过程最多可能需要 30 分钟才能完成。完成后，Console 将为现有用户分配适当的角色。

Cloud Volumes ONTAP 角色适用于 Amazon Web Service (AWS)、Microsoft Azure (Azure) 和 Google Cloud 环境。

[[support-for-new-ec2-instances-for-cloud-volumes-ontap-in-aws]]
=== 支持 AWS 中 Cloud Volumes ONTAP 的新 EC2 实例

Cloud Volumes ONTAP 9.16.1 及更高版本支持 M6id 和 M6idn EC2 实例系列，用于 AWS 中的新建和现有单节点和高可用性 (HA) 对部署。与第 5 代通用系统相比，这些第 6 代实例具有更快的处理能力、更强的网络功能和更高的性能。

* *M6id 实例*：通过高速、低延迟的本地 NVMe 存储提供计算和内存的平衡。所有 M6id 实例大小都支持 Flash Cache。这些实例非常适合数据记录和媒体处理等工作负载。
* *M6idn 实例*：提供与 M6id 实例相同的本地 NVMe 存储功能。所有 M6idn 实例大小都支持 Flash Cache。


我们不建议使用较小尺寸的 m6id.xlarge、m6id.2xlarge、m6idn.xlarge 和 m6idn.2xlarge 实例类型进行高性能或内存密集型工作负载。对于这些工作负载，请使用具有更高核心计数和更大内存的实例类型。

有关详细信息，请参阅 https://docs.netapp.com/us-en/cloud-volumes-ontap-relnotes/reference-configs-aws.html["AWS 中Cloud Volumes ONTAP支持的配置"^]。

[[flexible-billing-for-cloud-volumes-ontap-and-data-services-in-aws]]
=== AWS 中 Cloud Volumes ONTAP 和数据服务的灵活计费

NetApp 为您在 AWS 中的 Cloud Volumes ONTAP 订阅提供了增强的计费选项，让您可以更好地控制如何支付 Cloud Volumes ONTAP 容量和 NetApp 数据服务（如 NetApp Backup and Recovery）的费用。以前，所有 Cloud Volumes ONTAP 和数据服务的使用都是通过 AWS 市场中的 NetApp Intelligent Services (NIS) 列表一起计费的。

由于 AWS 企业折扣计划 (EDP) 的变化，NetApp 在 AWS 市场中引入了独立的 Cloud Volumes ONTAP 订阅，允许您将 Cloud Volumes ONTAP 容量的计费与数据服务分开。这种分离使您可以更灵活地管理 EDP 积分和成本。NetApp 建议使用独立订阅来更好地管理 Cloud Volumes ONTAP 使用情况的成本，同时在 2026 年 7 月 31 日 EDP 要求生效后继续使用 NIS 订阅进行数据服务计费。

AWS 中的独立 Cloud Volumes ONTAP 订阅仅支持基于容量的有效 Essentials 或 Freemium 许可证包。不支持 Professional 包。您可以选择最适合您需求的账单方式：

* *独立 Cloud Volumes ONTAP 订阅（推荐）*：通过 AWS 市场中的独立订阅支付您的 Cloud Volumes ONTAP 使用费用，同时继续通过 NIS 订阅收取数据服务费用。对于在 NetApp Console 中同时拥有与其 AWS 帐户相关联的独立订阅和 NIS 订阅的大多数客户，这是推荐的选项。
* *组合 NIS 订阅*：通过现有的 NIS 列表同时支付 Cloud Volumes ONTAP 和数据服务。组合方法适用于在 NetApp Console 中仅与其 AWS 帐户关联的 NIS 订阅的现有客户。但是，NetApp 建议您为 Cloud Volumes ONTAP 使用量设置计费首选项，并切换到独立订阅以更好地管理成本。


根据您的订阅类型，您将获得以下方案：

.仅限独立订阅（新客户）
如果您在 NetApp Console 中仅有一个与您的 AWS 账户关联的独立 AWS 订阅，并且没有 NIS 订阅，则可以在部署期间仅选择 Essentials 许可证包。您无法使用数据服务*备份和恢复*选项。

.独立订阅和 NIS 订阅（现有客户）
如果您有与 AWS 账户关联的 NIS 和独立订阅，并且尚未通过 NetApp Console 中的*计费首选项*选项卡设置计费首选项，则默认情况下，Cloud Volumes ONTAP 的使用将继续通过 NIS 计费。这也是运行 Professional 包的任何 Cloud Volumes ONTAP 实例的默认值，因为独立列表不支持 Professional。您可以设置计费首选项，并选择如何对每项服务计费：

* 将独立订阅设置为 Cloud Volumes ONTAP（使用 Essentials 包）的首选计费账户。如果您只有 Professional 包，则可以将其转换为 Essentials 包，并通过设置计费首选项对您的 Cloud Volumes ONTAP 使用量收费。我们推荐这种方法，因为它可以帮助您更有效地管理 Cloud Volumes ONTAP EDP 积分。
* 使用 NIS 订阅进行计费数据服务，或通过 Professional 包使用 Cloud Volumes ONTAP。在部署期间，启用 *Backup and Recovery* 数据服务，并选择 Professional 许可证包以通过 NIS 订阅向数据服务收费。


.仅限 NIS 订阅（现有客户）
如果您是拥有与 AWS 帐户关联的 NIS 订阅的现有客户，则适用此规则。您可以继续像现在一样同时计费 Cloud Volumes ONTAP 和数据服务。但是，NetApp 建议切换到独立订阅方法，以便更好地管理您的 Cloud Volumes ONTAP 使用情况：

* 订阅 AWS 门户上的新独立列表。
* 将独立订阅链接到您的 AWS 帐户。
* 使用 Essentials 包将计费首选项设置为 Cloud Volumes ONTAP 使用的独立订阅。
* （如适用）将您的 Professional 套餐转换为 Essentials 套餐并设置账单首选项。
* 通过您的 NIS 订阅继续计费数据服务。



NOTE: 当您使用多种订阅类型或在房源之间迁移时，不会出现重复的计量。

[[ability-to-manage-console-agent-proxy-settings-for-cloud-volumes-ontap]]
=== 能够管理 Cloud Volumes ONTAP 的 Console 代理设置

现在，即使失去连接或代理设置不正确，也可以在 NetApp Console 代理上管理 Cloud Volumes ONTAP 的代理设置。以前，如果 Console 代理在 20 分钟内无法连接到 Cloud Volumes ONTAP，它将使用默认设置覆盖您的手动代理设置。这导致通信失败，包括 AutoSupport 消息问题。若要保留现有系统的代理设置，请运行以下 API 调用：

`PUT /occm/config`

在请求正文中包含以下参数：

[source, json]
----
{
  "proxyMode": "No_Overwrites"
}
----
默认模式为标准模式，这意味着如果 Console 代理在 20 分钟内无法连接到 Cloud Volumes ONTAP，它将使用默认设置覆盖您的代理设置。

https://docs.netapp.com/us-en/console-automation/cvo/put-occm-config.html["配置可修改的 NetApp Console 参数"^]

[[cloud-volumes-ontap-support-for-next-generation-google-cloud-vms]]
=== Cloud Volumes ONTAP 支持下一代 Google Cloud 虚拟机

借助 9.18.1，NetApp 将新的 Cloud Volumes ONTAP 部署从 N2 虚拟机过渡到下一代 Google Cloud C3 系列虚拟机，以获得更快、更具可扩展性的体验。现在，您可以在 Google Cloud 中部署 Cloud Volumes ONTAP 9.18.1 及更高版本时利用 C3 系列虚拟机。C3 系列计算机通过使用 Google 虚拟网卡（gVNIC）和 Hyperdisk Balanced 磁盘，确保密集型工作负载的动态性能，从而提供更高的性能和更大的容量限制。


NOTE: 目前，Cloud Volumes ONTAP 仅在单节点部署中支持 C3 系列。

如果您的 Cloud Volumes ONTAP 系统运行 9.18.1 或更高版本，则用于轻松部署单节点的预配置包将自动使用 C3 虚拟机，同时使您能够根据工作负载需求自定义 IOPS 和吞吐量参数。同样，在创建聚合时，您可以添加 Hyperdisk Balanced 磁盘，以在 Google Cloud 中实现更好的性能和可扩展性。此外，您可以为默认 Flash Cache 支持选择 C3 系列计算机的 LSSD 变体。

将卷添加到聚合时，无法更改 C3 VM 的磁盘类型，因为 C3 仅支持 Hyperdisk Balanced 磁盘。同样，将具有 N2 VM 类型的系统复制到 C3 VM 时，磁盘类型默认设置为 Hyperdisk Balanced。

https://docs.netapp.com/us-en/cloud-volumes-ontap-relnotes/reference-configs-gcp.html["Google Cloud 中 Cloud Volumes ONTAP 支持的配置"^]

https://docs.cloud.google.com/compute/docs/general-purpose-machines#c3_series["Google 文档：C3 机器系列"^]



[[vnet-security-for-cloud-volumes-ontap-in-azure]]
=== Azure 中 Cloud Volumes ONTAP 的 VNet 安全性

Azure 单个和多个可用区域中的 Cloud Volumes ONTAP 9.18.1 及更高版本部署支持 Azure Virtual Network (VNet) 加密，作为其分层安全策略的一部分来保护传输中的数据。Cloud Volumes ONTAP 利用 Azure 的本机数据报传输层安全性 (DTLS) 协议来保护 ONTAP 节点、管理接口和其他 Azure 服务之间的通信，防止拦截和未经授权的访问。这种网络级加密是对 ONTAP 内置存储和静态数据保护的补充，可为您的数据提供端到端的安全性。

https://docs.netapp.com/us-en/storage-management-cloud-volumes-ontap/reference-networking-azure.html#networking-for-azure-vnet-encryption["用于 Azure VNet 加密的网络"]

[[larger-disks-in-google-cloud]]
=== Google Cloud 中的更大磁盘

Cloud Volumes ONTAP 现在支持 Google Cloud 中的 64 TB 磁盘。


NOTE: 由于 Google Cloud 的限制，仅使用磁盘的最大系统容量仍为 256 TB。



[[new-machine-types-in-google-cloud]]
=== Google Cloud 中的新机器类型

Cloud Volumes ONTAP 现在支持在 Google Cloud 中使用这些 PAYGO 许可证的以下机器类型：

* n2-standard-4 带有 Explore 许可证和 BYOL
* n2-standard-8 具有标准许可证和 BYOL
* 具有 Premium 许可证和 BYOL 的 n2-standard-32
