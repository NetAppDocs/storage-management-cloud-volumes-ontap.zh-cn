您可以使用 `Quick create`在 Amazon Web Services (AWS) 和 Microsoft Azure 中部署 Cloud Volumes ONTAP，以便在单节点和高可用性 (HA) 配置中实现更快的部署。与高级方法相比，这种简化流程通过在单个页面上自动设置默认值并最大限度地减少导航来减少部署步骤。

.步骤
. 从左侧导航菜单中，选择“存储”>“管理”。
. [[订阅]]在 Canvas 页面上，单击 *添加系统* 并按照提示进行操作。

.系统详细信息
. *云提供商帐户/凭据*：根据您的 Console 代理自动填写帐户和订阅详细信息。如果您有多个帐户或订阅，请选择要使用的帐户或订阅。如果 Console 代理不可用，系统将提示您 https://docs.netapp.com/us-en/console-setup-admin/concept-agent-install-options.html["创建控制台代理"^]。
. *名称*：输入 Cloud Volumes ONTAP 系统的名称（也称为集群名称）。NetApp Console 使用系统（集群）名称来命名 Cloud Volumes ONTAP 系统和 VM 实例。如果您选择该选项，它还会使用该名称作为预定义安全组的前缀。
. *ONTAP 凭据*：这些是 Cloud Volumes ONTAP 集群管理员帐户的凭据。您可以使用这些凭据通过 ONTAP System Manager 或 ONTAP CLI 连接到 Cloud Volumes ONTAP。您可以保留默认的 _admin_ 用户名或将其更改为自定义用户名。
. *标签*：标签是 AWS 或 Azure 资源的元数据。NetApp Console 将标签添加到 Cloud Volumes ONTAP 实例以及与该实例关联的每个资源。在创建 Cloud Volumes ONTAP 系统时，您最多可以从用户界面添加 15 个标签，然后您可以在创建后添加更多标签。有关更多信息，请参阅：

.部署和配置
. *部署类型/拓扑*：选择要使用的部署类型，单个节点、单个可用区 (AZ) 中的高可用性 (HA) 或多个 AZ 中的 HA。
. *网络配置*：输入您在区域、VNet 和子网的规划工作表中记录的网络信息。
+
.. *区域*：默认情况下，将选择与资源关联的云帐户的区域。
.. *可用性区域*：您可以根据您选择的部署类型以及所选区域是否支持可用性区域来选择可用性区域。
.. *子网*：选择区域的子网。
+
[role="tabbed-block"]
====
.对于 AWS
--
*VPC*：对于 AWS，请根据您的计划输入具有子网的区域的 VPC

.AWS 高可用性
如果选择了 HA 配置，请输入以下信息：

.单可用区高可用性
... *调解器访问*：指定调解器访问信息。调解器是一个单独的实例，用于监控 HA 对的健康状况并在发生故障时提供仲裁。提供密钥对名称以使中介实例能够连接到 AWS EC2 服务，并选择连接方法。


.多个可用区中的高可用性
... *可用区域和中介*：选择每个节点的可用区域 (AZ) 以及要部署Cloud Volumes ONTAP HA 对的中介和相应子网。
... *浮动 IP*：如果您选择多个 AZ，请为 NFS 和 CIFS 服务以及集群和 SVM 管理指定浮动 IP 地址。 IP 地址必须位于该区域内所有 VPC 的 CIDR 块之外。有关更多详细信息，请参阅link:https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/reference-networking-aws.html#requirements-for-ha-pairs-in-multiple-azs["多个可用区中Cloud Volumes ONTAP HA 的 AWS 网络要求"^]。
... *调解器访问*：指定调解器访问信息。调解器是一个单独的实例，用于监控 HA 对的健康状况并在发生故障时提供仲裁。提供密钥对名称以使中介实例能够连接到 AWS EC2 服务，并选择连接方法。
... *路由表*：如果您选择了多个 AZ，请选择包含到浮动 IP 地址的路由的路由表。如果您有多个路由表，则选择正确的路由表非常重要。否则，某些客户端可能无法访问Cloud Volumes ONTAP HA 对。有关路由表的更多信息，请参阅 http://docs.aws.amazon.com/AmazonVPC/latest/UserGuide/VPC_Route_Tables.html["AWS 文档：路由表"^]。

.充电和服务
. *许可证*：选择要用于此 Cloud Volumes ONTAP 系统的许可证类型。您可以从 Professional、Essentials 和 Freemium 许可证中进行选择。有关不同许可证的信息，请参阅 link:concept-licensing.html["了解Cloud Volumes ONTAP许可证"^]。
. *数据服务和功能*：保持服务启用或禁用您不想与Cloud Volumes ONTAP一起使用的服务。
+
** https://docs.netapp.com/us-en/data-services-data-classification/concept-classification.html["了解有关NetApp分类的更多信息"^]
** https://docs.netapp.com/us-en/data-services-backup-recovery/concept-backup-to-cloud.html["了解有关NetApp Backup and Recovery的更多信息"^]
** link:concept-worm.html["了解Cloud Volumes ONTAP上的 WORM 存储"]
+

TIP: 如果您想利用 WORM 和数据分层，则必须禁用备份和恢复并部署版本 9.8 或更高版本的Cloud Volumes ONTAP系统。



. * NetApp支持站点帐户*：如果您有多个帐户，请选择要使用的帐户。
. 检查或编辑您输入的详细信息，然后单击*创建*。


ifdef::azure[]

在 _Azure_ 中，*概述*部分显示部署过程完成时将分配的资源。

endif::azure[]

ifdef::aws[]

在 _AWS_ 中，创建 Cloud Volumes ONTAP 系统后，NetApp Console 立即在指定的 VPC 中启动测试实例以验证连接。如果成功，Console 会立即终止实例，然后开始部署系统。如果 Console 无法验证连接，则系统创建失败。测试实例可以是 `t2.nano`（对于默认 VPC 租赁）或 `m3.medium`（对于专用 VPC 租赁）。

endif::aws[]

.完成后
控制台部署Cloud Volumes ONTAP系统。您可以在审核页面上跟踪进度。

如果您在部署 Cloud Volumes ONTAP 系统时遇到任何问题，请查看故障消息。您还可以在系统上选择*重新创建环境*，以使用 `Advanced`模式向导继续进行部署。


CAUTION: 部署过程完成后，请勿在 AWS 或 Azure 云门户中修改系统生成的 Cloud Volumes ONTAP 配置，尤其是系统标记。对这些配置所做的任何更改都可能导致意外行为或数据丢失。
