Google Cloud 中支持多种 Cloud Volumes ONTAP 配置。



[[supported-disk-types-in-google-cloud]]
== Google Cloud 中支持的磁盘类型

除了存储用户数据之外，NetApp 还为 Google Cloud 中的系统数据购买云存储。



[[internal-disks-for-system-data-in-single-node-systems]]
=== 单节点系统中用于系统数据的内部磁盘

* 一个 10 GiB SSD 永久磁盘，用于存储启动数据
* 一个 64 GiB SSD 持久磁盘，用于存储根数据
* 一个 500 GiB SSD 持久磁盘，用于NVRAM
* 一个 315 GiB 标准持久磁盘，用于保存核心
* 启动和根数据的快照
+

NOTE: 重启时会自动创建快照。

* 默认情况下，启动磁盘和根磁盘是加密的。




[[internal-disks-for-system-data-in-ha-pairs]]
=== HA 对中系统数据的内部磁盘

* 两个 10 GiB SSD 持久磁盘用于启动数据
* 四个 64 GiB SSD 持久磁盘用于根数据
* 两个 500 GiB SSD 持久磁盘用于NVRAM
* 两个 315 GiB 标准持久磁盘，用于保存核心
* 一个 10 GiB 标准持久磁盘，用于存储中介数据
* 一个 10 GiB 标准永久磁盘，用于中介启动数据
* 启动和根数据的快照
+

NOTE: 重启时会自动创建快照。

* 默认情况下，启动磁盘和根磁盘是加密的。




[[google-cloud-storage-for-user-data]]
=== Google Cloud 用户数据存储

在 Google Cloud 中，聚合最多可以包含 6 个大小相同的磁盘。最大磁盘大小为 64 TiB。

磁盘类型可以是_区域 SSD 持久磁盘_、_区域平衡持久磁盘_或_区域标准持久磁盘_。您可以将永久性磁盘与 Google 存储桶配对，以link:concept-data-tiering.html["低成本对象存储"]。

Cloud Volumes ONTAP 支持 Hyperdisk Balanced 磁盘，适用于 9.18.1 及更高版本的单节点部署，以及 9.19.1 及更高版本的单可用区 (AZ) 中的高可用性 (HA) 配置。



[[supported-disk-sizes]]
=== 支持的磁盘大小

在 Google Cloud 中，一个聚合最多可包含 6 个类型和大小相同的磁盘。支持以下磁盘大小：

* 100 GB
* 500 GB
* 1 TB
* 2 TB
* 4 TB
* 8 TB
* 16 TB
* 64 TB




[[supported-configurations-by-license]]
== 按许可证支持的配置

Cloud Volumes ONTAP 在 Google Cloud 中作为单节点系统和高可用性 (HA) 节点对提供，用于容错和无中断操作。

不支持将单节点系统升级到 HA 对。如果要在单节点系统和 HA 对之间切换，则需要部署新系统并将数据从现有系统复制到新系统。

Cloud Volumes ONTAP 可以在云提供商提供的保留或按需 VM 实例上运行。不支持使用其他 VM 实例类型的解决方案。

[[flash-cache-support]]
== Flash Cache 支持

从 Cloud Volumes ONTAP 9.13.0 开始，_Flash Cache_、高写入速度和 8,896 字节的更高最大传输单元 (MTU) 可用于以下 HA 对部署实例：

* n2-标准-16
* n2-标准-32
* n2-标准-48
* n2-标准-64


部署符合条件的实例类型时，可以启用 Flash Cache 和高写入速度。要启用 8,896 字节的更高最大传输单元，必须选择 VPC-1、VPC-2 或 VPC-3 进行部署。更高的 MTU 可实现更高的网络吞吐量。有关启动其中一个部署的更多信息，请参见 https://docs.netapp.com/us-en/bluexp-cloud-volumes-ontap/task-deploying-gcp.html["在 Google Cloud 中启动 Cloud Volumes ONTAP"^]。


NOTE: Flash Cache、高写入模式和 8,896 的 MTU 依赖于功能，并且不能在配置的实例中单独禁用。

Cloud Volumes ONTAP 9.18.1 及更高版本支持 C3 系列虚拟机的本地 SSD (LSSD) 变体，适用于单节点部署；Cloud Volumes ONTAP 9.19.1 及更高版本支持 C3 系列虚拟机的本地 SSD (LSSD) 变体，适用于单个 AZ 中的 HA 部署。对于这些部署，如果选择 C3 系列虚拟机的本地 SSD (LSSD) 变体，则默认情况下会启用 Flash Cache。即使目标变体支持 Flash Cache，也无法在 Cloud Volumes ONTAP 部署中的 C3 LSSD 虚拟机变体之间切换。
